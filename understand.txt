📁 MAIN FOLDERS & WHAT THEY CONTAIN
================================================================================

/src/
└─ Main source code folder - ALL your application code lives here

    /src/app/
    ├─ page.tsx          → MAIN HOMEPAGE - Imports all sections and displays them
    ├─ layout.tsx        → ROOT LAYOUT - Sets up fonts, themes, metadata, favicon
    └─ globals.css       → GLOBAL STYLES - CSS variables, dark/light theme colors, 
                           dot patterns, noise overlay

    /src/components/
    └─ All React components broken into subfolders

        /src/components/layout/
        ├─ Navbar.tsx        → Top navigation bar with logo, links, theme toggle
        ├─ Footer.tsx        → Bottom footer section with links
        ├─ ScrollIndicator.tsx → Shows scroll progress bar at top
        └─ Section.tsx       → Wrapper component for each section

        /src/components/sections/
        ├─ Hero.tsx          → Main hero/landing section with profile photos
        ├─ Projects.tsx      → Projects showcase section
        ├─ Experience.tsx    → Work experience timeline section
        ├─ Skills.tsx        → Skills grid section
        └─ Contact.tsx       → Contact form section

        /src/components/ui/
        ├─ ProfilePhotos.tsx  → Your 2 overlapping profile photos (Hero section)
        ├─ ProjectCard.tsx    → Individual project card component
        ├─ SkillCard.tsx      → Individual skill card
        ├─ SkillBadge.tsx     → Skill badge/pill component
        ├─ ExperienceItem.tsx → Single experience entry
        ├─ StatCounter.tsx    → Animated counter (5+, 10+, etc)
        ├─ InterestTag.tsx    → Interest tags (Backend Dev, AI Agents, etc)
        ├─ RotatingTags.tsx   → Rotating animation tags
        ├─ ThemeToggle.tsx    → Dark/Light theme toggle button
        ├─ BrandIcon.tsx      → Brand/company icon
        ├─ GradientMesh.tsx   → Gradient background mesh
        ├─ NoiseOverlay.tsx   → Noise texture overlay
        ├─ DotPattern.tsx     → Dot pattern background
        └─ SkillCategory.tsx  → Container for skill badges

        /src/components/providers/
        └─ ThemeProvider.tsx  → Wraps entire app with dark/light theme support

    /src/hooks/
    ├─ useScrollProgress.ts  → Hook to detect scroll position
    ├─ useInView.ts         → Hook to detect if element is in viewport
    └─ useCounter.ts        → Hook for animated counter numbers

    /src/lib/
    ├─ data.ts             → ⭐ ALL SITE CONTENT (projects, experience, skills, bio)
                             EDIT THIS FILE TO UPDATE YOUR PORTFOLIO CONTENT
    ├─ types.ts            → TypeScript interfaces/types for all data
    ├─ animations.ts       → Framer Motion animation definitions
    ├─ fonts.ts            → Google Fonts imports (Playfair, Sora, JetBrains Mono)
    └─ utils.ts            → Helper functions (cn() for Tailwind classes)

/public/
└─ Static files served directly to browser

    /public/images/
    ├─ profile-1.jpg       → Your 1st profile photo (Hero section)
    ├─ profile-2.jpg       → Your 2nd profile photo (Hero section)
    ├─ web-projects-cover.jpg    → Cover image for Web Projects
    ├─ ai-projects-cover.jpg     → Cover image for AI Projects
    ├─ logo.jpg            → Your logo (appears in navbar top-left)
    └─ combineapps-logo.svg → Company logo (Experience section)

    favicon.ico            → Browser tab icon (appears in browser tab)
    og-image.png           → Social media preview image (for sharing links)
    noise.svg              → Noise texture SVG file

/api/
└─ Currently EMPTY - Future location for backend API endpoints

/database/
└─ Currently EMPTY - Future location for database schemas

/services/
└─ Currently EMPTY - Future location for service integrations

🚀 HOW TO RUN & DEPLOY
================================================================================

DEVELOPMENT:
$ npm run dev
→ Starts dev server at http://localhost:3000
→ Auto-refreshes when you save files

PRODUCTION BUILD:
$ npm run build
$ npm run start

LINT CODE:
$ npm run lint

DEPLOYMENT:
Push to main branch → Auto-deploys to Vercel

================================================================================
📊 COMPONENT HIERARCHY (How they connect)
================================================================================

/src/app/page.tsx (MAIN PAGE)
│
├─ imports sections in order:
│
├─ → HeroSection (shows profile photos, name, title, stats)
│   └─ uses ProfilePhotos.tsx, StatCounter.tsx, InterestTag.tsx
│
├─ → ProjectsSection (shows project cards)
│   └─ uses ProjectCard.tsx for each project
│
├─ → ExperienceSection (shows experience timeline)
│   └─ uses ExperienceItem.tsx for each job
│
├─ → SkillsSection (shows skills grid)
│   └─ uses SkillCategory.tsx → SkillCard.tsx
│
└─ → ContactSection (shows contact form)
    └─ uses ContactForm component

All sections wrapped with:
├─ Navbar.tsx (top fixed navigation)
├─ ScrollIndicator.tsx (progress bar)
├─ Footer.tsx (bottom footer)
└─ NoiseOverlay.tsx (background texture)

Theme managed by:
└─ ThemeProvider.tsx (wraps everything)

================================================================================
🆘 COMMON QUESTIONS
================================================================================

Q: Where do I change my name?
A: /src/lib/data.ts → heroData.name

Q: How do I add a new project?
A: /src/lib/data.ts → projectCategories array → add new object

Q: Where do I put my images?
A: /public/images/ folder

Q: How do I change colors/theme?
A: /src/app/globals.css → CSS variables (--background, --foreground, etc)

Q: Where are animations defined?
A: /src/lib/animations.ts → Framer Motion variants

Q: How do I add a new section?
A: Create /src/components/sections/NewSection.tsx → add data to data.ts → import in page.tsx

Q: Why isn't my image showing?
A: Make sure file path in data.ts exactly matches /public/images/filename.ext

Q: How do I deploy?
A: Push to main branch → Vercel auto-deploys
